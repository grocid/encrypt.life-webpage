The details

Encryption

The current implementation supports authenticated encryption. Suppose Alice want to send Bob a message. She fetches Bob's public key and encrypts a secret with it, which becomes a token. Then, she uses the token to derive a two secret keys, A and B. The key A is used to encrypt the message using a symmetric cipher (currently, this cipher is AES-256). The key B is used to create a message authentication code (MAC) using Poly1305. The ciphertext then becomes the tag, the ciphertext and the MAC.

Decryption

Alice now wants to decrypt. Using her private key, she decrypted the encrypted token and derives all necessary keys, i.e., A and B. Using these keys, she can verify the MAC and decrypt the message.
For more details, see the Github repository.
